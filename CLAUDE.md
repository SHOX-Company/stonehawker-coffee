## WAT Architecture (Adapted)

### Layer 1: Workflows

Use workflows/ for repeatable systems such as:

- homepage_build.md
- product_page_system.md
- mobile_optimization.md
- conversion_audit.md
- deployment_checklist.md
- media_optimization.md

Each workflow should define:

- Objective
- Inputs
- Files touched
- Steps
- Expected output
- Edge cases

### Layer 2: Agents

You are the orchestrator.

Responsibilities:

- Inspect current files before editing
- Make targeted upgrades
- Avoid breaking working sections
- Improve UX continuously
- Ask clarifying questions only when necessary
- Think like an owner
- Ship progress daily

### Layer 3: Tools

Tools may include:

- HTML/CSS/JS edits
- Build tools
- Image compression
- Video optimization
- Deployment scripts
- Shopify integrations
- Analytics installs
- SEO setup

Only build custom scripts when they save recurring time or unlock leverage.

## How To Operate

### Before Editing Anything

1. Read current file structure
2. Understand existing layout
3. Preserve working systems
4. Improve weakest bottleneck first
5. Check mobile experience first

### When Making Changes

Prefer:

- Surgical edits
- Clean structure
- Reusable components
- Mobile responsiveness
- Premium visuals
- Conversion improvements
- Fast load speed

Avoid:

- Random rewrites
- Bloated frameworks
- Breaking CSS chains
- Unnecessary dependencies
- Large uncompressed media
- Style over substance
